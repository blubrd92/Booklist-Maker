## Detailed behavior specs

### firebase-init.js

At the top, check the hostname. If it's the public tool or local dev without the override query param, export nulls and return. Otherwise, initialize the Firebase app with the config, initialize Firestore, initialize Auth, and export the auth and db instances. Set persistence to local (browserLocalPersistence) so sessions survive browser restarts. Expose window.firebaseAuth and window.firebaseDb for use by other modules.

### library-config.js

Imports from firebase-init.js. If firebaseAuth or firebaseDb is null (public tool, not a branded instance), set window.LIBRARY_CONFIG = null and dispatch a custom event 'library-config-ready'.

Otherwise, derive the libraryId from the hostname subdomain (or from the ?library= query param on localhost).

Try to fetch the config from the libraries-public collection first using getDoc(doc(db, 'libraries-public', libraryId)). If the document exists, this is an unauthenticated public instance. Store the data in window.LIBRARY_CONFIG, set window.LIBRARY_REQUIRES_AUTH = false, dispatch 'library-config-ready', and skip all auth logic. The user is never shown a login modal.

If the libraries-public document does not exist, this is a gated instance. Set window.LIBRARY_REQUIRES_AUTH = true and dispatch 'library-config-needs-auth'. The auth.js module listens for this event and shows the login modal. After successful sign-in, library-config.js fetches from the libraries collection using getDoc(doc(db, 'libraries', libraryId)). On success, store the data in window.LIBRARY_CONFIG and dispatch 'library-config-ready'. On failure (permission denied, document doesn't exist, network error), dispatch 'library-config-failed' with the error in the event detail.

The schema for both libraries-public and libraries documents is the same:

- displayName: string, e.g. "San Rafael Public Library"
- primaryColor: hex string, e.g. "#003366"
- accentColor: hex string, e.g. "#cc0000"
- defaultCoverFont: CSS font-family string, e.g. "'Oswald', sans-serif"
- defaultBookFont: CSS font-family string, e.g. "'Calibri', sans-serif"
- brandingImagePath: string path, e.g. "assets/img/libraries/sanrafael/logo.png"
- defaultCoverLayout: string, one of "classic", "masonry", "staggered", "tilted"
- defaultExtendedMode: boolean

You don't need to create any documents. They will be created manually in the Firebase console in a later session. For now, just make sure the loader knows how to read them.

### auth.js

Imports from firebase-init.js. If firebaseAuth is null, do nothing.

Listen for the 'library-config-needs-auth' event. When it fires, render the login modal. Don't render it on page load, only when explicitly told to via this event. This way, public instances (libraries-public configs) never show the modal at all.

The modal covers the entire viewport and prevents interaction with the tool underneath. It contains:

- A heading that says "Sign in to Booklister" (don't include the library name yet, since we don't know it until after sign-in)
- An email input
- A password input
- A "Sign in" button
- A "Forgot password?" link that triggers Firebase's sendPasswordResetEmail flow
- An error message area for failed sign-in attempts

On successful sign-in, hide the modal. library-config.js will then fetch the gated config and dispatch 'library-config-ready'. On failed sign-in, show a friendly error message: "Incorrect email or password" for auth/wrong-password and auth/user-not-found, "Too many attempts, try again later" for auth/too-many-requests, "Network error, check your connection" for network failures, generic "Sign-in failed, please try again" for anything else.

Add a sign-out button to the existing app header, only visible when window.LIBRARY_REQUIRES_AUTH is true and the user is signed in. Place it next to the existing tour button. Use a small icon button matching the existing header button style. On click, call signOut and reload the page.

The modal should be styled to feel like part of the tool: same fonts, same color palette, same border radius and shadow conventions as the existing modals. Look at .modal-overlay and .modal-content in styles.css for reference. Put the new styles in assets/css/auth.css and link it from index.html.

### Integration with app.js (step four)

Listen for the 'library-config-ready' event. When it fires, if LIBRARY_CONFIG is not null, apply the values from the config:

- Set the document title to LIBRARY_CONFIG.displayName + " Booklister"
- Update the .header-credit text to "for " + LIBRARY_CONFIG.displayName  
- Replace the default branding image source with LIBRARY_CONFIG.brandingImagePath
- Set the default font for the cover header to LIBRARY_CONFIG.defaultCoverFont (only if no other font is currently selected, don't override user choices)
- Set the default font for book blocks similarly
- Set the selected layout to LIBRARY_CONFIG.defaultCoverLayout
- Set the extended mode toggle to LIBRARY_CONFIG.defaultExtendedMode

These should happen before the user has a chance to interact with the tool. If the event fires after the tool is already initialized (which it can, since auth and Firestore reads are async), apply the changes and trigger any necessary re-renders.

If LIBRARY_CONFIG is null (public tool), do nothing. Tool behaves exactly as before.
