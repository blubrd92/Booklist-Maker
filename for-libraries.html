Booklister for Libraries

Booklister is free for individual librarians to use, but libraries as institutions sometimes want something more formal: their own branded version, predictable support, and a clear point of contact for their staff. This page describes the institutional offer.

What you get

A custom-branded instance of Booklister. Your library's name and logo become the starting point, along with a default for whether book descriptions auto-draft as staff add titles. When your staff open the tool, it's already set up the way you want, so every booklist they make starts on-brand.

A dedicated subdomain. Something like libraryname.booklister.org. It's yours, it's stable, and it points at a version of the tool set up for your library specifically.

Priority support and maintenance. Throughout the contract period I'll keep the tool running, fix bugs, and push improvements as they come up. Questions from custom-instance libraries jump ahead of the general weekends-and-evenings queue that handles inquiries to the public tool. Booklister is a side project around my day job, so I can't commit to a specific response window, but I prioritize custom-instance issues and don't let them sit.

Your library manages its own staff accounts. A designated admin at your library adds and removes staff through a dedicated admin console without needing to go through me. You stay in control of your own roster, and I remain available as a backstop for anything the admin UI doesn't cover.

Your feedback shapes the tool. If your library needs something Booklister doesn't do yet, let me know. Anything I build goes into the main tool and rolls out to every instance. I don't build one-off features for individual libraries because I'd rather keep one Booklister than maintain a dozen slightly different versions.

AI-drafted book descriptions, included. Booklister has a one-click "magic wand" on each book that drafts a short reader's advisory summary using a live web search plus a language model. This feature costs real money per use (API charges), so it's reserved for custom library instances. On the free public tool at booklister.org the button shows a notice saying so; on your branded instance it's fully enabled, and I cover the underlying API costs on your behalf. Your library never needs to set up or manage any third-party API accounts.

Public-tool fallback. If the custom instance ever has an outage, your staff can fall back to the public tool at booklister.org without any configuration changes. The core tool is the same; only the branding and the AI description feature differ.

Setup takes about a week. Most of that time is waiting for DNS propagation and procurement paperwork rather than development. Once your library is ready to proceed, spinning up the branded instance itself takes an afternoon.

What it costs

Pricing is quoted per library. The underlying costs (API usage and support time) scale with how many staff use the tool and how heavily they use it, so a small branch library and a large county system shouldn't land at the same number. Send me a note with your library's size and expected usage, and I'll put together a quote.

Catalog integrations

If your library wants Booklister to connect directly to its catalog system, that's possible but handled as a separate scoped project. The complexity varies a lot depending on the system and what you need, so pricing is scoped to each project individually. If this is something you're interested in, get in touch and we can walk through what it would involve.

Ending the relationship

If your library ever decides to stop using Booklister, winding things down is straightforward. Send me a note from the contact page and I'll confirm in writing, then remove your library's Firestore configuration, all of its staff memberships, and the branding assets associated with the instance. I don't keep backup copies after a deletion request is completed, so anything you still want (saved .booklist files, exported PDFs) should be downloaded before the takedown. This is a manual process I handle personally, same as account creation. I aim to complete deletion requests within one business day of confirmation.

How to get started

The best first step is a conversation. If you're curious whether Booklister makes sense for your library, head over to the contact page and send me a note. I'd rather talk through your library's actual workflow and needs than ask you to commit to anything sight unseen.
